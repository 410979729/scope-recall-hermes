# 自动经验提取与遗忘机制实施计划

> 目标：让 scope-recall 不再依赖 Joy 人工复审大量记忆，而是由插件/玉衡从任务轨迹中自动提取可复用经验手册，并自动归档、合并、隐藏低价值或过期记忆。

## 核心判断

当前已有底座：

- `journal_entries`：原始任务轨迹。
- `memories`：普通长期记忆。
- `task_episodes`：任务片段表，当前未写入。
- `procedural_playbooks`：可复用经验手册表，当前未自动写入。
- `experience_runs`：经验使用记录，当前只有手动工具会写。

缺口：

1. 没有从 `journal_entries` 自动切出“任务片段”。
2. 没有从成功任务片段自动生成“可复用经验手册”。
3. 没有低风险自动启用机制。
4. 没有默认软遗忘/归档机制。
5. 现有维护口径偏“人工复审”，不适合 Joy 的实际使用方式。

## 设计原则

1. **用户不做记忆管理员**：Joy 不需要看大量日志、工具轨迹或候选记忆。
2. **先证据后晋升**：没有测试、验证、成功信号的任务，不自动启用。
3. **低风险自动启用**：只读、文档、普通本地检查等低风险流程，证据充分即可启用。
4. **高风险谨慎使用**：重启服务、远程运维、删除文件、仓库推送、凭据相关流程，即使生成经验手册，也必须带现场核验和授权边界。
5. **默认软遗忘**：优先归档/隐藏/标记替代，而不是直接物理删除。
6. **硬删除只处理垃圾和敏感内容**：工具噪声、明确重复、疑似密钥泄漏可以硬删除。
7. **普通记忆只存锚点**：长流程进入可复用经验手册，普通记忆只保留简短索引。

## 第一阶段：最小闭环

### 任务 1：新增自动经验提取模块

文件：

- 新建：`experience_promotion.py`
- 测试：`tests/test_experience_promotion.py`

功能：

1. 按可访问范围读取 `journal_entries`。
2. 按 `session_id` 聚合任务轨迹。
3. 检测成功信号：`passed`、`通过`、`ok`、`green`、`验证`、`完成`。
4. 检测验证信号：`pytest`、`ruff`、`doctor`、`release gate`、`smoke`、`测试`。
5. 检测风险信号：`push`、`commit`、`restart`、`delete`、`rm`、`token`、`password`、`密钥`、`重启`、`删除`、`推送`。
6. 自动写入 `task_episodes`。
7. 自动创建“候选经验手册”。
8. 低风险且证据充分时自动启用。
9. 高风险时保留候选/需要再次代理复核，不要求 Joy 人工看。

验收：

- 给定一段包含用户目标、工具调用、测试通过、最终总结的 journal，能生成 1 个任务片段和 1 个经验手册。
- 低风险经验手册状态为 `promoted`。
- 含 `push` / `restart` / `token` 的经验手册不直接启用。
- 重复运行不会重复创建同一份经验手册。

### 任务 2：新增遗忘机制模块

文件：

- 新建：`forgetting.py`
- 测试：`tests/test_forgetting.py`

功能：

1. 生成遗忘报告。
2. 识别运行时包装噪声、助手草稿、极短低价值记忆、重复记忆、过期候选。
3. 默认只做软归档：在 `metadata` 中写入：
   - `lifecycle: archived`
   - `forget_reason`
   - `archived_at`
   - `superseded_by`（重复记忆适用）
4. 可选硬删除只用于明确垃圾/敏感内容。
5. 已归档记忆不重复归档。

验收：

- dry-run 不修改数据库。
- apply 后候选记忆被标记为 archived。
- 重复记忆只保留一条，其余标记 superseded。
- 默认不物理删除普通记忆。

### 任务 3：新增工具入口

文件：

- 修改：`schemas.py`
- 修改：`tooling.py`
- 修改：`provider.py`
- 测试：`tests/test_experience_tools.py`

新增工具：

- `scope_recall_experience_promote`：运行自动经验提取。
- `scope_recall_forgetting_report`：只读遗忘报告。
- `scope_recall_forgetting_run`：执行遗忘，默认 dry-run。

要求：

- 这些入口默认维护工具权限保护。
- 报告字段使用中性结构，不暴露敏感内容。
- 文档叙述使用中文“经验手册/遗忘/归档”，英文工具名只作为机器接口存在。

## 第一阶段不做的事

- 不自动推仓库。
- 不自动重启 Hermes。
- 不默认在后台自动启用经验晋升。
- 不物理删除普通长期记忆。
- 不把大段工具输出写入经验手册。
- 不让 Joy 人工逐条复审。

## 最终验收样例

输入：一段类似“scope-recall 发布检查”的任务记录，包含版本核对、测试通过、发布检查通过、医生检查通过、未获推送授权。

期望输出：

1. 自动生成任务片段。
2. 自动生成经验手册，标题类似：`scope-recall 发布候选收口流程`。
3. 经验手册步骤包含：版本核对、测试、代码检查、发布检查、医生检查、授权边界。
4. 因包含推送/发布风险，默认不直接启用，或启用但禁止直接执行推送，只允许提醒授权边界。
5. 下次任务前经验预检能召回这份经验手册。
